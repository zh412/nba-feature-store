# NBA Feature Store Data Model

This document describes the data warehouse structure used by the NBA Feature Store pipeline.

The warehouse consists of two tables:

- `pr_see_player_dimension`
- `pr_see_daily_player_game_log`

The dimension table stores stable player metadata, while the feature store table contains player-level game statistics.

---

# Player Dimension Table

`pr_see_player_dimension`

Stores player attributes that do not change on a per-game basis.

Columns:

| Column | Description |
|------|-------------|
| PLAYER_ID | Unique NBA player identifier |
| POSITION | Primary player position |
| HEIGHT | Player height |
| EXP | Years of NBA experience |

This table is built automatically by the pipeline using:

- `commonallplayers`
- `commonplayerinfo`

The dimension table is joined during ingestion to enrich player game records with stable player metadata.

---

# Player Game Feature Store

`pr_see_daily_player_game_log`

This is the primary analytics table generated by the pipeline.

Each row represents:

**one player in one NBA game**

The table contains **90 player-level features** derived from multiple NBA Stats API endpoints.

---

# Column Groups

The feature store columns can be grouped into several logical categories.

### Player Identity

- PLAYER_ID
- PLAYER_NAME
- TEAM_ID
- TEAM_TRICODE

---

### Game Metadata

- GAME_ID
- GAME_DATE
- GAME_STATUS
- GAME_TIME_UTC
- ARENA_NAME
- ARENA_CITY
- ARENA_STATE
- ARENA_COUNTRY
- ATTENDANCE

---

### Player Metadata (Dimension Join)

- POSITION
- HEIGHT
- EXP

---

### Traditional Box Score Statistics

Examples:

- fieldGoalsMade
- fieldGoalsAttempted
- reboundsTotal
- assists
- steals
- blocks
- turnovers
- foulsPersonal
- points
- plusMinusPoints

These metrics come from:

`boxscoretraditionalv3`

---

### Advanced Metrics

Examples:

- offensiveRating_ADV
- defensiveRating_ADV
- netRating_ADV
- trueShootingPercentage_ADV
- effectiveFieldGoalPercentage_ADV
- usagePercentage_ADV
- reboundPercentage_ADV

These metrics come from:

`boxscoreadvancedv3`

---

### Usage Metrics

Examples:

- usagePercentage_USAGE
- percentageFieldGoalsMade_USAGE
- percentageAssists_USAGE
- percentagePoints_USAGE

These metrics come from:

`boxscoreusagev3`

---

### Team Context Metrics

Examples:

- effectiveFieldGoalPercentage_TEAM
- teamTurnoverPercentage_TEAM
- offensiveReboundPercentage_TEAM

These metrics come from:

`boxscorefourfactorsv3`

---

### Warehouse Metadata

- ROW_KEY
- INGESTED_AT_UTC
- LOAD_BATCH_ID

These columns are generated by the ingestion pipeline to support:

- idempotent loads
- ingestion auditing
- debugging and monitoring

---

# Partitioning

The feature store is partitioned by:

`GAME_DATE`

This allows efficient time-series queries and prevents full table scans.

---

# Clustering

The table is clustered by:

- PLAYER_ID
- TEAM_ID

This improves query performance for player and team level analytics.
